## 환경변수 세팅

이 문서는 로컬/개발/배포 환경에서 환경변수를 설정하는 방법을 정리한다.

### 기본 원칙

- 민감 정보는 `.env.local`에만 둔다.
- 공용/문서화 목적 값은 `.env.example`로 관리한다.
- 클라이언트에 노출되는 값은 `NEXT_PUBLIC_` 접두어를 사용한다.
- 서버 전용 값은 `NEXT_PUBLIC_` 없이 사용한다.

### 파일 구성

- `.env.local` : 로컬 개발 전용(개인별)
- `.env` : 공통 기본값(선택)
- `.env.development` : 개발 환경 전용
- `.env.production` : 프로덕션 전용
- `.env.example` : 필수 키 목록 템플릿

### 로컬 설정 절차

1. `.env.example`을 복사해 `.env.local`을 만든다.
2. 각 변수 값을 본인 환경에 맞게 채운다.
3. 앱을 실행한다.

```shell
cp .env.example .env.local
```

### 클라이언트/서버 구분

- 브라우저에서 사용해야 하는 값만 `NEXT_PUBLIC_`을 붙인다.
- 서버 전용 값은 API 호출, 인증/보안 로직 등에서만 사용한다.

### 변경 시 체크리스트

- 신규 변수 추가 시 `.env.example`에 반드시 반영
- 문서의 변수 설명 업데이트
- 민감 정보가 Git에 올라가지 않도록 확인

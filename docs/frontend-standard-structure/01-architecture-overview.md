# 아키텍처 개요

## 설계 목표

- 도메인 책임을 명확히 분리해 변경 범위를 최소화한다.
- UI, 상태, 비즈니스 로직의 경계를 유지한다.
- 페이지 단위가 아닌 "기능/도메인" 중심으로 구조를 설계한다.

## 레이어 개요(FSD 기반)

- `app`: 전역 설정, 레이아웃, 라우팅 엔트리
- `pages`: 라우팅 단위의 화면 조합
- `widgets`: 화면 조립 단위의 복합 UI
- `features`: 사용자 행동 단위의 기능
- `entities`: 도메인 모델과 관련 UI/로직
- `shared`: 공통 유틸, UI, 훅, 타입

## 설계 원칙 요약

- 상위 레이어는 하위 레이어에 의존 가능, 역방향 의존 금지.
- 도메인 규칙은 `features` 또는 `entities`에서 관리.
- 재사용 가능성은 `shared`로 승격하되, 과도한 승격은 금지.

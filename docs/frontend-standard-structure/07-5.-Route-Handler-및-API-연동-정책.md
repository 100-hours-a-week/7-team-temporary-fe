## 5. Route Handler 및 API 연동 정책

### 5.1 Route Handler 사용 범위

- Route Handler는 인증 토큰 중계 및 접근 제어를 위한 프레임워크 레벨 역할에 한정한다.
- 비즈니스 규칙은 Route Handler에 포함하지 않는다.

### 5.2 api-routes 미사용 정책

- Next.js Route Handlers는 파일 시스템 기반 라우팅을 따르므로,
  복잡한 서버 로직을 직접 구현하면 FSD 레이어 구조와 충돌할 수 있다.
- 본 프로젝트에서는 `api-routes` 세그먼트를 도입하지 않는다.
- `app/api` 디렉터리를 라우팅 및 프록시 용도로만 유지한다.

### 5.3 백엔드 연동 책임 분리

- 비즈니스 로직은 백엔드 서버에서 처리한다.
- 프론트는 라우팅/프록시/인증 토큰 중계 역할에 집중한다.
- Transport는 `apiFetch` 단일 진입점으로 통일하고, endpoint는 `shared/api/endpoints`로 관리한다.

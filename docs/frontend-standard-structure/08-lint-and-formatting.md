# Lint / Formatting 정책

## 도입 도구

- ESLint를 기본 검사 도구로 사용한다.
- Prettier를 포맷 기준으로 사용한다.

## ESLint 적용 기준

- 레이어 규칙과 네이밍 규칙을 우선 적용한다.
- 경고/오류 기준을 명확히 문서화한다.

## 규칙 관리 원칙

- 규칙 변경은 팀 합의 후 반영한다.
- 규칙 예외는 최소화하고 근거를 남긴다.

## 자동화

- 커밋/PR 파이프라인에서 린트 실행을 기본화한다.

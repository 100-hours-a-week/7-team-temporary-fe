## 3. Next.js(App Router) 적용 전략

### 3.1 app 디렉터리 역할 정의

- `app/`는 Next.js App Router 전용 엔트리로 사용한다.
- `src/app`은 제거하고 `app/`로 통합한다.
- `app/(public)` / `app/(protected)`로 접근 레벨을 표현한다.

### 3.2 Route Group 설계 기준

- 역할: “이 화면은 public/protected다”를 폴더 구조로 표현한다.
- 하지 않는 것: 인증 판단 로직(실행 로직)
- `app/**/page.tsx`는 라우팅 어댑터 역할만 수행한다.

### 3.3 layout 설계 기준

- 역할: 보호 영역 공통 UI(shell) + 2차 방어(SSR 렌더링 단계 안전장치)
- 하지 않는 것: 페이지별 도메인 로직
- 페이지별 비즈니스 로직은 `src/pages`로 제한한다.

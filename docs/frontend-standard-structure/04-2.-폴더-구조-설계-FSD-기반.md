## 2. 폴더 구조 설계 (FSD 기반)

### 2.1 레이어 구성 개요

- shared
- entities
- features
- widgets
- pages

### 2.2 레이어별 책임 정의

- shared: 전역 공통 유틸/스타일/API/쿼리 레이어
- entities: 도메인 핵심 엔티티 모델 및 저수준 API
- features: 사용자 액션 단위 기능
- widgets: 복합 UI 블록(여러 기능 조합)
- pages: 화면 단위 조합 및 라우팅 대상 컴포지션
- 공통 UI라도 **비즈니스 규칙이 섞이면 feature/widget로 이동**한다.

### 2.3 레이어 간 의존성 규칙

- 하위 레이어는 상위 레이어를 참조하지 않는다.
- `shared → entities → features → widgets → pages` 단방향 규칙을 유지한다.
- 아키텍처 기준은 [FSD 아키텍처 레이어 정의 및 배치 기준]()을 따른다.

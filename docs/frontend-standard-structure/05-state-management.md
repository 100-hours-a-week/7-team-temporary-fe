# 상태 관리 구조

## 서버 상태 관리 전략

- 서버 상태는 전용 도구로 관리하고 캐시를 일관되게 유지한다.
- 데이터 갱신 정책과 무효화 기준을 문서화한다.

## 전역 UI 상태 관리 전략

- 전역 UI 상태는 최소 범위에서만 관리한다.
- 도메인 상태와 UI 상태의 책임을 분리한다.

## 상태 간 책임 경계

- 서버 상태와 UI 상태를 혼합하지 않는다.
- 비즈니스 규칙은 도메인 레이어에서 관리한다.

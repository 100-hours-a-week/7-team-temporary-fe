# 라우팅 및 레이아웃

## 라우팅 원칙

- App Router 기준으로 페이지를 구성한다.
- 라우트는 목적 기반으로 분리하고, 중복 경로를 금지한다.
- 화면 구성은 `pages`에서 조합하고, `widgets`로 분리한다.

## 레이아웃 구성 기준

- 전역 레이아웃은 `app`에서만 정의한다.
- 페이지별 레이아웃은 라우트 스코프에서 구성한다.
- 공통 레이아웃 컴포넌트는 `shared/ui`로 이동한다.

## 화면 플로우 기준(요약)

- 로그인/회원가입 등 주요 플로우는 문서화하고, 검증 기준을 명시한다.
- 검증 규칙은 UI와 분리해 재사용 가능하도록 한다.

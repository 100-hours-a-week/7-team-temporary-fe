## 1. 아키텍처 개요 (Architecture Overview)

### 1.1 전체 구조 요약

- 본 프로젝트는 **Next.js(App Router) + React 기반의 클라이언트 애플리케이션**이다.
- **FSD(Feature-Sliced Design)** 아키텍처를 기반으로 폴더 구조와 책임 경계를 정의한다.
- Next.js 라우팅 엔트리는 `app/`, 화면 조합은 `src/pages`로 분리한다.

### 1.2 설계 목표

- 기능 수가 많고(Planner, Report, Chat, Auth 등) 도메인이 분리되어 있어
  **기능 단위 확장성**과 **변경 영향 범위 최소화**가 중요하다.
- 화면 단위(Page) 중심 구조는 장기적으로 결합도가 높아지고,
  공통 로직 재사용과 테스트가 어려워진다.

### 1.3 핵심 설계 원칙

- 전역 규칙 → 공통 유틸 → 도메인 → 페이지 순으로 **의존성 단방향 규칙**을 강제한다.
- 서버 상태는 TanStack Query, 전역 UI 상태는 Zustand로 역할을 분리한다.
- 라우팅 엔트리와 화면 조합 책임을 분리해 변경 영향 범위를 최소화한다.

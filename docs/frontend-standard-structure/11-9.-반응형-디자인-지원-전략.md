## 9. 반응형 디자인 지원 전략

### 9.1 설계 원칙

- **모바일 퍼스트 + 반응형 레이아웃**을 따른다.
- 레이아웃 책임은 Page/Widget 레벨에서 관리한다.

### 9.2 Breakpoint 기준

- TailwindCSS 기반 브레이크포인트를 사용한다.

### 9.3 구현 기준

- 고정 픽셀 값 지양, 상대 단위 사용
- 컴포넌트 내부에서 반응형 분기 처리

### 9.4 애니메이션 / 모션 기준

#### 사용 목적 분류

- **피드백용 모션**
  - 버튼 클릭, 토글, hover, focus
  - → `react-bits` 또는 CSS transition 사용

- **상태 전환 모션**
  - 리스트 진입/퇴장, 로딩 → 결과 전환
  - → `react-bits` (단일 요소, 짧은 지속)

- **맥락 전달용 모션**
  - 스크롤 연동, 시퀀스형 강조, 스토리텔링
  - → `GSAP` 사용

#### 구조적 책임

- **Page**
  - 페이지 진입/이탈, 라우트 전환 모션
- **Widget**
  - 레이아웃 단위의 집합적 모션
- **Feature**
  - 사용자 액션에 반응하는 상태 전환 모션
- **Shared**
  - 재사용 가능한 모션 프리미티브(easing, duration)

#### 성능 및 접근성 기준

- `transform`, `opacity` 중심으로 구성한다.
- `height`, `width`, `top/left` 기반 애니메이션은 금지한다.
- `prefers-reduced-motion` 활성 시:
  - duration을 0 또는 최소값으로 처리한다.
- 모바일/저사양 환경에서는:
  - 스크롤 연동 및 연속 모션을 비활성화한다.

#### Next.js 제약

- 애니메이션 훅은 `"use client"` 컴포넌트에서만 사용한다.
- 서버 컴포넌트에서는 모션 로직을 포함하지 않는다.

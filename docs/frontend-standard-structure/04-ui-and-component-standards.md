# 공통 UI 및 컴포넌트 배치 기준

## 공통 UI 정의 기준

- 재사용 범위가 명확한 컴포넌트를 공통 UI로 승격한다.
- 도메인 규칙이 섞이면 `features` 또는 `entities`로 이동한다.

## 도메인 조건 전달 원칙

- 공통 UI는 도메인 상태를 직접 알지 않는다.
- 필요한 값은 props로 전달하고, 이벤트는 콜백으로 전달한다.

## 비즈니스 규칙 포함 UI 기준

- 비즈니스 규칙이 포함되면 공통 UI로 분리하지 않는다.
- UI와 로직은 분리하고, 테스트 가능한 구조를 유지한다.

## 스타일링 우선순위(요약)

- 공통 스타일 규칙을 최우선으로 적용한다.
- 예외가 필요한 경우 최소 범위에서만 허용한다.
